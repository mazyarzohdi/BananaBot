# 🍌 BananaBot — ربات تلگرام فروش و مدیریت VPN (+ پنل وب و سیستم نمایندگی)

ربات تلگرامی کامل برای فروش و مدیریت خودکار سرویس‌های VPN روی پنل **3x-ui**؛ همراه با
یک **پنل وب (Telegram Mini App)**، **سیستم نمایندگی (Reseller)** کامل، **تایید خودکار
پرداخت از روی پیامک بانک**، **بکاپ‌گیری خودکار زمان‌بندی‌شده**، **سیستم تیکت پشتیبانی**،
**سیستم معرفی (رفرال)**، و **تمدید خودکار سرویس از کیف پول**.

## ✨ امکانات

### 👤 کاربر (داخل ربات)
- خرید سرویس با ایجاد خودکار کانفیگ روی پنل + کوپن تخفیف
- اکانت تست رایگان
- مشاهده‌ی سرویس‌ها، لینک کانفیگ، لینک ساب، مصرف، و بروزرسانی لینک
- **تمدید سرویس** (دستی) و **تمدید خودکار از کیف پول** (روشن/خاموش برای هر سرویس، با اطلاع‌رسانی خودکار در صورت کمبود موجودی)
- **یادآور انقضا** — قبل از تمام‌شدن سرویس، پیام یادآوری خودکار دریافت می‌کند
- کیف پول و افزایش موجودی (کارت‌به‌کارت با تایید دستی ادمین، یا **تایید کاملاً خودکار از روی پیامک بانک**)
- **سیستم معرفی/رفرال** — لینک اختصاصی، پاداش نقدی به ازای اولین شارژ موفق زیرمجموعه
- **سیستم تیکت پشتیبانی** داخل ربات (علاوه بر تماس مستقیم با آیدی پشتیبانی، که ادمین می‌تواند روشن/خاموش کند)
- **پنل نمایندگی** — خرید پلن نمایندگی، ساخت کانفیگ برای مشتریان خودش از حجم تخصیصی، مدیریت از داخل ربات یا پنل وب
- FAQ و آموزش

### 🔧 ادمین (داخل ربات)
- مولتی ادمین
- **آمار مالی واقعی** — درآمد امروز/هفته/ماه/کل، پرفروش‌ترین محصولات، درآمد نمایندگی‌ها
- **خروجی CSV** از لیست کاربران و پرداخت‌ها
- مدیریت پنل‌ها (افزودن، تست اتصال، لیست Inbound)
- مدیریت کامل محصولات (افزودن/ویرایش/فعال‌غیرفعال/حذف)
- تأیید/رد پرداخت‌های کارت‌به‌کارت (با ثبت اپراتور تاییدکننده)
- جستجوی کاربر، تغییر موجودی، بن/آن‌بن، **یادداشت اختصاصی روی هر کاربر**
- **مدیریت کامل نمایندگان** — مشاهده، غیرفعال/فعال‌سازی، ویرایش دستی حجم و تاریخ انقضا، و **حذف کامل نماینده** (با پاکسازی خودکار کانفیگ‌های او از روی پنل X-UI)
- مدیریت پلن‌های نمایندگی
- **مدیریت تیکت‌های پشتیبانی** — مشاهده، پاسخ، بستن
- ارسال همگانی (Broadcast) با کنترل نرخ ارسال (Throttle) و مدیریت خودکار محدودیت سیل تلگرام
- مدیریت کوپن تخفیف (درصدی/مبلغ ثابت، محدودیت تعداد استفاده)
- مدیریت FAQ و آموزش‌ها
- تنظیمات کامل (متن خوش‌آمد، کانال اجباری، اکانت تست، حداقل شارژ، سیستم معرفی، یادآور انقضا، تایید خودکار پرداخت، بکاپ خودکار، و ...)

### 🌐 پنل وب (Telegram Mini App)
یک پنل تحت وب کامل که هم برای کاربران عادی (داخل خودِ تلگرام، به‌عنوان Mini App) و هم برای
ادمین در دسترس است:
- ورود با **Telegram Login Widget** (مرورگر) یا خودکار از طریق **Mini App** (داخل تلگرام)
- کاربر: مشاهده‌ی سرویس‌ها، خرید، کیف پول، پنل نمایندگی
- ادمین: مدیریت کاربران، محصولات، پنل‌ها، پرداخت‌ها (با پاداش خودکار رفرال هنگام تایید)، پلن‌های نمایندگی، نمایندگان، و تنظیمات — دقیقاً همان قابلیت‌هایی که داخل ربات هست، از طریق مرورگر هم در دسترس‌اند

### 🤝 سیستم نمایندگی (Reseller)
- ادمین پلن‌های نمایندگی (حجم کل + مدت + قیمت) تعریف می‌کند
- نماینده پلن را می‌خرد و از حجم تخصیصی‌اش، برای مشتریان خودش کانفیگ می‌سازد (هم از داخل ربات، هم از پنل وب)
- ادمین در هر لحظه می‌تواند حجم/تاریخ‌انقضای هر نماینده را دستی اصلاح کند یا نماینده را کامل حذف کند

### 🔁 تایید خودکار پرداخت (از روی پیامک بانک)
یک سرویس systemd کاملاً مستقل (`bananabot-webhook`) که پیامک واریزی بانک را (از طریق یک
اپ فورواردر SMS روی گوشی ادمین) دریافت، مبلغ را با پرداخت‌های در انتظار تطبیق، و در عرض
چند ثانیه بدون دخالت ادمین موجودی کاربر را شارژ می‌کند. راهنمای کامل راه‌اندازی:
[`AUTO_PAYMENT_SETUP.md`](AUTO_PAYMENT_SETUP.md)

### 💾 بکاپ‌گیری خودکار زمان‌بندی‌شده
یک حلقه‌ی پس‌زمینه‌ی داخل خودِ ربات که طبق فاصله‌ی زمانی تنظیم‌شده توسط ادمین (مثلاً هر ۲۴
ساعت)، هم از دیتابیس خودِ ربات و هم از دیتابیس هر پنل X-UI متصل بکاپ می‌گیرد، آن‌ها را برای
همه‌ی ادمین‌ها در تلگرام ارسال می‌کند، توی `data/backups/` ذخیره می‌کند (سازگار با منوی
ریستور `manage.sh`)، و بکاپ‌های قدیمی‌تر از حد تنظیم‌شده را خودکار پاک می‌کند.

## 🚀 نصب سریع (توصیه‌شده)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/mazyarzohdi/BananaBot/main/install.sh)
```

یا دستی:

```bash
git clone https://github.com/mazyarzohdi/BananaBot.git
cd BananaBot
sudo bash install.sh
```

اسکریپت نصب به‌صورت خودکار:
1. ✅ پیش‌نیازهای سیستم را نصب می‌کند
2. ✅ محیط مجازی Python می‌سازد و کتابخانه‌ها را نصب می‌کند
3. ✅ توکن ربات، آیدی ادمین، شماره کارت و کانال اجباری را از شما می‌پرسد و `.env` می‌سازد
4. ✅ دیتابیس را با اسکیمای کامل می‌سازد (`db_schema.py`)
5. ✅ سرویس systemd ربات (`bananabot`) را ایجاد و راه‌اندازی می‌کند
6. ✅ در صورت تمایل، پنل وب (Mini App، سرویس `bananabot-web`) را هم با دامنه/SSL راه‌اندازی می‌کند

## ⚙️ پیکربندی (`.env`)

فایل `.env` در ریشه‌ی پروژه (کنار `main.py`) ساخته می‌شود:

```env
# توکن ربات از @BotFather
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ

# آیدی عددی ادمین (یک یا چند عدد با کاما)
ADMIN_IDS=123456789,987654321

# مسیر پایگاه داده
DATABASE_PATH=data/bot.db

# زبان پیش‌فرض (فعلاً فقط fa پشتیبانی می‌شود)
DEFAULT_LANG=fa

# شماره کارت برای پرداخت کارت‌به‌کارت (اختیاری)
CARD_NUMBER=6037-XXXX-XXXX-XXXX
CARD_HOLDER=نام صاحب کارت

# کانال اجباری برای استفاده از ربات (اختیاری)
REQUIRED_CHANNEL=@mychannel

# آدرس عمومی پنل وب — این خط را خودِ manage.sh هنگام راه‌اندازی پنل وب
# (گزینه‌ی [17]) می‌سازد و می‌نویسد؛ نیازی به تنظیم دستی نیست.
PANEL_URL=https://panel.example.com/panel
```

> تنظیمات مربوط به **تایید خودکار پرداخت** (فعال/غیرفعال، کلید امنیتی، پورت وبهوک)، **سیستم
> معرفی**، **یادآور انقضا**، و **بکاپ خودکار** در فایل `.env` نیستند — همه از داخل منوی
> «⚙️ تنظیمات» ربات (یا `admin/settings/` در پنل وب) و به‌صورت زنده قابل تغییرند.

## 🎛️ اسکریپت مدیریت (`manage.sh`)

پس از نصب، برای مدیریت کامل ربات:

```bash
sudo bash /opt/BananaBot/manage.sh
```

بالای منو همیشه وضعیت زنده‌ی هر سه سرویس (ربات / پنل وب / وبهوک پرداخت خودکار) نمایش داده
می‌شود. امکانات منو:

| بخش | گزینه‌ها |
|-----|----------|
| **کنترل ربات** | روشن/خاموش/ریستارت، مشاهده‌ی لاگ زنده، ۵۰ خط آخر لاگ |
| **تنظیمات** | تغییر توکن، آیدی ادمین، شماره کارت، کانال اجباری، مشاهده‌ی تنظیمات فعلی |
| **عملیات پیشرفته** | **آپدیت کامل ربات از گیت‌هاب** (با توقف امن هر سه سرویس قبل از تغییر دیتابیس، بکاپ ایمنی خودکار، و بررسی صحت قبل/بعد از آپدیت)، **آپدیت خودِ منوی مدیریت (`manage.sh`)** به‌تنهایی، حذف کامل ربات |
| **دیتابیس** | بکاپ دستی، ریستور از بکاپ (با پاکسازی خودکار فایل‌های WAL باقیمانده و بررسی صحت)، بررسی/تعمیر خودکار اسکیما |
| **پنل وب** | وضعیت، روشن/خاموش/ریستارت، راه‌اندازی/تنظیم مجدد (دامنه، پورت، مسیر، SSL) |
| **وبهوک پرداخت خودکار** | وضعیت، ریستارت، **توقف**، **مشاهده‌ی لاگ زنده**، **۵۰ خط آخر لاگ** |

نکته‌ی امنیتی: «آپدیت کامل ربات از گیت‌هاب» پیش از هرگونه تغییر، خودکار هر سه سرویس را
متوقف می‌کند (چون آپدیت اسکیمای دیتابیس هم‌زمان با سرویس‌های زنده می‌تواند دیتابیس را خراب
کند)، یک بکاپ ایمنی می‌گیرد، و در پایان دقیقاً همان سرویس‌هایی که قبلاً روشن بودند را دوباره
روشن می‌کند.

## 📡 API پنل 3x-ui

برای فعال‌سازی API در پنل:
1. وارد پنل 3x-ui شوید
2. به **Settings → Security** بروید
3. **API Token** را فعال کنید و کپی کنید (همین توکن هنگام افزودن پنل در ربات وارد می‌شود)

| عملیات | Endpoint |
|--------|----------|
| ایجاد کلاینت | `POST /panel/api/clients/add` |
| ویرایش/بروزرسانی کلاینت | `POST /panel/api/clients/update/{id}` |
| حذف کلاینت | `POST /panel/api/clients/delete/{id}` |
| ریست مصرف کلاینت | `POST /panel/api/clients/resetTraffic/{email}` |
| دریافت اطلاعات کلاینت | `GET /panel/api/clients/{email}` |
| دریافت لینک‌های ساب | `GET /panel/api/clients/links/{email}` |
| مصرف کلاینت | `GET /panel/api/clients/traffic/{email}` |
| بکاپ دیتابیس پنل | `GET /panel/api/server/getDb` |
| لیست Inbound | `GET /panel/api/inbounds/options` |

احراز هویت: `Authorization: Bearer <API_TOKEN>`

## 🔄 دستورات سریع systemd

```bash
# وضعیت هر سه سرویس
systemctl status bananabot bananabot-web bananabot-webhook

# شروع / توقف / ریستارت ربات
systemctl start bananabot
systemctl stop bananabot
systemctl restart bananabot

# لاگ زنده‌ی هرکدام
journalctl -u bananabot -f
journalctl -u bananabot-web -f
journalctl -u bananabot-webhook -f
```

> ترجیحاً به‌جای دستورات بالا از منوی `manage.sh` استفاده کنید — مخصوصاً برای آپدیت و ریستور
> بکاپ، چون منو مراقب توقف/راه‌اندازی امن سرویس‌ها و گرفتن بکاپ ایمنی هم هست.

## 📝 نکات مهم

- برای **On-Hold** (فعال‌شدن سرویس فقط پس از اولین اتصال کاربر)، هنگام افزودن پنل فیلد `on_hold` را فعال کنید.
- فایل `.env` پس از نصب فقط توسط `root` قابل خواندن است (chmod 600).
- پایگاه‌داده‌ی SQLite در مسیر `data/bot.db` ذخیره می‌شود و در حالت WAL اجرا می‌شود (چون هم‌زمان سه پروسه — ربات، پنل وب، وبهوک — به آن دسترسی دارند).
- همه‌ی هندلرهای بخش ادمین پشت بررسی سطح دسترسی (`@admin_only`) قرار دارند؛ فقط آیدی‌های ذکرشده در `ADMIN_IDS` به آن‌ها دسترسی دارند.
- تایید پرداخت (چه دستی توسط ادمین، چه خودکار از روی پیامک بانک) به‌صورت atomic و امن در برابر race condition پیاده‌سازی شده — تایید هم‌زمان یک پرداخت از دو مسیر مختلف هرگز باعث شارژ دوبل نمی‌شود.
- برای آپدیت یا ریستور بکاپ، همیشه از `manage.sh` استفاده کنید (نه دستورات git/cp دستی) — این‌ها مراقب توقف امن سرویس‌ها و گرفتن بکاپ ایمنی خودکار هستند.
